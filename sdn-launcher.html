SDN Project Setup

Path Tracker Runner

Step-by-step guide to upload and run your 3 SDN files on GitHub Codespaces — a free Linux terminal in your browser.

Your 3 files
Simulator
sdn-path-tracker.py

Path Tracer

Standalone simulator. Runs without Mininet. Traces paths, shows flow table, simulates link failures.

Controller
controller.py

Ryu Controller

Real SDN controller using Ryu framework. Handles packet-in events and installs flow rules.

Topology
topo.py

Mininet Topology

Defines 3 hosts + 3 switches with multiple paths. Used with Mininet for real network simulation.

Setup steps
1

Create a free GitHub account

Go to github.com and sign up. It's free. Skip if you already have one.

Open github.com/signup ↗
2

Create a new repository

Click the + icon → New repository. Name it sdn-path-tracker. Set it to Public. Check "Add a README file". Click Create repository.

Open github.com/new ↗
3

Upload your 3 files

Inside your new repo, click Add file → Upload files. Drag and drop all 3 files: sdn-path-tracker.py, controller.py, topo.py. Click Commit changes.

4

Open GitHub Codespaces (free Linux terminal)

On your repo page, click the green Code button → Codespaces tab → Create codespace on main. Wait ~30 seconds for it to load. A full VS Code terminal opens in your browser.

5

Install dependencies in the terminal

In the bottom terminal panel, run:

pip install networkx eventlet==0.30.2 ryu
6

Run Option A — Standalone Simulator (Easiest)

No Mininet needed. Just run:

python3 sdn-path-tracker.py

Then choose from the menu: trace path (H1→H5), show flow table, simulate failures.

7

Run Option B — Full Ryu + Mininet (Two terminals)

First install Mininet in Codespaces:

sudo apt-get install -y mininet

Open Terminal 1 — start Ryu controller:

ryu-manager controller.py --observe-links

Open Terminal 2 (click + in terminal panel) — start Mininet:

sudo mn --custom topo.py --topo mytopo --controller remote --mac

Inside Mininet prompt test with:

pingall
8

Clean up when done

Always run this after exiting Mininet to avoid errors next time:

sudo mn -c
